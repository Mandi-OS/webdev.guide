Workspace
All sites have 3 axes XYZ.
  X - width(1D), Y - height(2D), Z - level index(3D).
You ask what is the layer, and why 3D if I see everything in 2D - flat?
- I answer: All elements have their own height position. Imagine a box in which we we threw a bunch of soft toys, closed them and we have only one transparent wall.
Now compress the box, and when we look through the transparent wall inside it, we will see just stacked together "layers" of stuffed toys.
In the case of a website, this wall is the browser screen, and our elements are soft toys. This is how it looks like:

By default, all elements have an auto-setting - auto. However, we can also set our own value through CSS: z-index: [level]; or JavaScript: 
element.zIndex = [level];
element.style.zIndex = "[level]"; // Through styles
Mainly we use it in headers, footers, navs and dropdown menus.
Server-client conception
Apart from the usual local area networks, this concept is present everywhere. Its essence lies in the exchange of messages between the server and the client.
Client is the user himself, and what happens only at him, locally, without going to the server.
Server is also the "control manager", it gives out instructions to all clients and stores data.
There is also often a "mailman" between the Server and the Client, or Hosting - one of the most famous being "Cloudflare".
Hosting can be compared to a checkpoint. It also tries to provide a secure connection(entry\exit) and often has protection against hacker attacks.
For example, on the bank page, when we go to the "Transfer History" tab, the client asks the server for this history, then processes it with a script and adds it to the list we see.
The server can return any response to the Client, but it is not certain that the Client will be able to process it correctly. Usually developers try to maximize compatibility and also add error handling.
Compilation
Now about how the browser compiles our code. The browser compiles(scans, understands and executes) the site in this sequence:
Any document that we link through the <head> section has the option of using the flag defer and async. 
<script type="text/javascript" src="/webdevelop.guide/storage/script1.js" defer></script>
<script type="text/javascript" src="/webdevelop.guide/storage/script2.js" async></script>
Defer means to run the script after the site compilation(i.e. HTML and CSS loading) and in the order in which all scripts with defer section <head> are written.
Async means launching the script immediately after it is loaded, regardless of the site compilation stage and position in the queue.
 Async in contrast to defer is used much less frequently.
But the use of defer is very important, because most sites are self-reproducing, that means they build themselves, and the defer is often an indispensable element of the work.
