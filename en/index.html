Workspace
All sites have 3 axes XYZ.
  X - width(1D), Y - height(2D), Z - layer level(3D).
You ask what is the layer, and why 3D if I see everything in 2D - flat?
- I answer: All elements have their own height position. Imagine a box in which we we threw a bunch of soft toys, closed them and we have only one transparent wall.
Now compress the box, and when we look through the transparent wall inside it, we will see just stacked together "layers" of stuffed toys.
In the case of a website, this wall is the browser screen, and our elements are soft toys. This is how it looks like:

By default, all elements have an auto-setting - auto. However, we can also set our own value through CSS: z-index: [layer-level]; or JavaScript: 
element.zIndex = [layer-level];
element.style.zIndex = "[layer-level]"; // Through styles
Mainly we use it in headers, footers, navs and dropdown menus.
Server-client conception
Apart from the usual local area networks, this concept is present everywhere. Its essence lies in the exchange of messages between the server and the client.
Client is the user himself, and what happens only at him, locally, without going to the server.
Server is also the "control manager", it gives out instructions to all clients and stores data.
There is also often a "mailman" between the Server and the Client, or Hosting - one of the most famous being "Cloudflare".
Hosting can be compared to a KPP. It also tries to provide a secure connection(entry\exit) and often has protection against hacker attacks.
For example, on the bank page, when we go to the "Transfer History" tab, the client asks the server for this history, then processes it with a script and adds it to the list we see.
The server can return any response to the Client, but it is not certain that the Client will be able to process it correctly. Usually developers try to maximize compatibility and also add error handling.

 Compilation
Now about how the browser compiles our code. The browser compiles the site in this sequence:
Any document that we link through the <head> section has the option of using the flag defer and async. 
<script type="text/javascript" src="/webdevelop.guide/storage/script1.js" defer></script>
<script type="text/javascript" src="/webdevelop.guide/storage/script2.js" async></script>
Defer means to run the script after the site compilation(i.e. HTML and CSS loading) and in the order in which all scripts with defer section <head> are written.
Async means launching the script immediately after it is loaded, regardless of the site compilation stage and position in the queue.
Async in contrast to defer is used much less frequently.
But the use of defer is very important, because most sites are self-reproducing, and the defer is often an indispensable element of the work. Debugging
 All browsers have a debugging window, in browsers based on Chromium it is DevTools, in Firefox it is "Developer Tools". Often this window can be opened by pressing F12 or in the context menu, such as "Explore", "Page source code", etc.  Usually there are these tabs:
  • The whole compilated site source code
  • Styles
  • Console
  • Sources
  • Site fetches monitoring(Network)
  • Performance monitoring
  • Memory
  • Cash(App)
  • DOM

Dictionary Here are all details about the concepts sorted by topics.
Basic
  • Compilation - the process of converting source code into machine code, or code understandable to a compiler application (e.g. a browser).
  • Flag - variable usually equal to true(true) or false(false). For example: 
    let drinkCoffeeAtMornings = false;
if (drinkCoffeeAtMornings) {
	Sanya.drink("Coffee");
} else {
	Sanya.say("Damn diet!")
}
    If the flag is true(same as 1) then Sanya will drink coffee, and if the flag is false(or 0) then Sanya will complain about his hard life.
  • Self-reproducing sites - many sites are arranged in such a way that initially their page is empty, and there are only blanks. After that it is already filled with content, using scripts, and less often server injections. For example, on this site, the list of all articles at the top is populated automatically using JavaScript.
  • Server injection is code that is invisibly (or almost invisibly) inserted into an HTML page. For example, using PHP, the server can inject certain code depending on whatever it is.
  • Core(Chromium) - the base on which the following application code is built. Adapted to the basic needs of the application, it eases the future programming. For example, in the case of Chromium, it is the base for some browsers, such as Google Chrome and Microsoft Edge.
 DevTools
  • Inspector or Elements - Shows the compiled HTML-structure of the current page along with applied styles, scripts and attributes.
  • Styles - Here you can view and edit all stylesheets, and in addition At-rules.
  • Console - To execute commands in real time, output errors and logs.
  • Debugger or Sources - All connected files and source code maps. In text documents you can set break points for debugging.
  • Network - Allows you to track all requests from the site: their type, status, load time, request and response.
  • Performance - To record and analyze page performance, including FPS, CPU load, rendering latency. For example, you can start recording, then start loading the game, stop recording, and see how it loads the system.
  • Memory - Used for analyzing memory leaks, tracking consumption, and cleaning up unnecessary objects. Rarely used.
  • App - Allows you to view the entire cache: Local Storage, Session Storage, IndexedDB(very large local storage), Services and Cookies.
  • DOM - Full access to the generated DOM structure - how the browser sees our site.
